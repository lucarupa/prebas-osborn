### CU-01: Subir pieza creativa (asset)

**Actor principal:** Diseñador (`designer`)  
**Precondición:** El proyecto y la agencia existen; el usuario tiene rol `designer`.  
**Flujo principal:**
1. El diseñador envía `POST /assets` con los datos de la pieza y el archivo de la primera versión.
2. El sistema crea el registro `asset` en estado `pending_review` con `current_version = 1`.
3. El sistema crea el registro `asset_version` con `version_number = 1`.
4. Se notifica a los revisores/clientes asignados al proyecto (futuro).
5. El sistema retorna el asset y la versión creados.

**Postcondición:** El asset existe en la base de datos en estado `pending_review`.
