### CU-05: Consultar historial de aprobaciones de una pieza

**Actor principal:** Cualquier usuario con acceso al proyecto.  
**Precondición:** El asset existe.  
**Flujo principal:**
1. El usuario consulta `GET /assets/{id}`.
2. El sistema retorna el asset, todas sus versiones ordenadas, y todas las aprobaciones/rechazos.

**Postcondición:** El usuario visualiza el historial completo de decisiones y versiones.
