### CU-02: Agregar nueva versión a una pieza existente

**Actor principal:** Diseñador (`designer`)  
**Precondición:** El asset existe y su estado es `pending_review` o `rejected`. El usuario es el creador original o tiene permiso explícito.  
**Flujo principal:**
1. El diseñador envía una solicitud con el nuevo archivo y notas de la versión.
2. El sistema calcula el siguiente `version_number` (máximo actual + 1).
3. Se crea el nuevo `asset_version`; el asset actualiza `current_version` y vuelve a `pending_review`.
4. Se notifica a revisores/clientes.

**Postcondición:** Existe una nueva versión; las anteriores permanecen inmutables.
